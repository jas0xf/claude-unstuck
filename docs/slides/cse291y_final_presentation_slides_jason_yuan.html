What broken looks like06 / 19
The gap that never resumes.
sse_tap.py — cycle 1 · IPv6 · 200 · cf=…-LAX
+1721ms message_start … ( S I L E N C E ) … 180.5 s client gives up & disconnects n_events = 1 stop_reason = null healthy cycle: n_events = 30 byte-proof: up 1,120,748 B · down 0 B · 120 s
4 TCP signatures — all AFTER the upload is fully ACKed

① no-response

Upload ACKed, then zero bytes come back. Pure silence.

② partial

message_start arrives, then the stream stalls forever.

③ reset (RST)

Server sends a TCP RST — kills the socket at 60 / 120 / 170 s.

④ cutoff (FIN)

Connection FIN'd mid-stream — a clean close, but with no answer.

RST = TCP reset: the server abruptly tears down the connection.   FIN = orderly close, but it lands before any answer is sent.
